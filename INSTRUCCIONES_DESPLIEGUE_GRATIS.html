📋 Próximos Pasos

1

Crear Repositorio en GitHub

Sube tu código a GitHub para que Render pueda acceder a él: 

git init
git add .
git commit -m "Initial commit - Librería JODA"
git branch -M main
git remote add origin https://github.com/TU_USUARIO/libreria-joda.git
git push -u origin main

📦 Crear Repositorio en GitHub

2

Crear Cuenta en Render.com

100% Gratuito - Sin tarjeta de crédito requerida

  • Ve a Render.com y regístrate con GitHub
  • Autoriza el acceso a tus repositorios
  • No necesitas ingresar información de pago

🎯 Crear Cuenta en Render

3

Desplegar Backend en Render

En el Dashboard de Render:

  1. Click en "New +" → "Web Service"
  2. Conecta tu repositorio libreria-joda
  3. Configuración:
    • Name: libreria-joda
    • Environment: Python 3
    • Region: Oregon (más cercano)
    • Branch: main
    • Build Command: pip install -r requirements.txt
    • Start Command: gunicorn -w 1 -b 0.0.0.0:$PORT run:app
    • Plan: Free
  4. Click "Create Web Service"

🚀 Crear Web Service

4

Configurar Variables de Entorno

En la página de tu Web Service → Pestaña "Environment":

Variable Valor
FLASK_ENV production
SECRET_KEY (cualquier texto aleatorio largo)
5

Esperar Despliegue (5-10 minutos)

Render construirá y desplegará tu aplicación automáticamente. Verás los logs en tiempo real. Una vez completado, obtendrás una URL como: 

https://libreria-joda.onrender.com
6

Conectar Firebase con Render

Actualiza firebase.json con la URL de Render: 

{
  "hosting": {
    "public": "public",
    "rewrites": [{
      "source": "/**",
      "destination": "https://libreria-joda.onrender.com"
    }]
  }
}

Luego vuelve a desplegar Firebase: 

firebase deploy --only hosting
7

Importar Tus Datos

Una vez que el sistema esté en línea:

  1. Accede a https://libreria-joda.web.app
  2. Inicia sesión con:
    • Usuario: admin
    • Contraseña: admin123
  3. Ve a Configuración → Importar Datos
  4. Sube el archivo datos_backup_20260203_171155.json

⚡ Importante - Limitaciones del Plan Gratuito

  • Sleep automático: El servicio en Render se duerme después de 15 minutos sin actividad. La primera carga puede tardar 30-60 segundos.
  • Solución: Usa UptimeRobot (gratis) para hacer ping cada 5 minutos y mantenerlo activo.
  • Base de datos SQLite: Se reinicia con cada despliegue. Mejor usa PostgreSQL gratuito de Render para persistencia.

💡 Mejora: Base de Datos PostgreSQL Persistente

Para evitar perder datos con cada despliegue:

  1. En Render Dashboard → "New +" → "PostgreSQL"
  2. Plan: Free (1 GB)
  3. Copia la DATABASE_URL
  4. Agrégala como variable de entorno en tu Web Service
  5. Actualiza config/config.py para usar PostgreSQL en producción
